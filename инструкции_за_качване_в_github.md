# Инструкции за качване на проект в GitHub

## 1. Създаване на ново репозитори в GitHub
1. Влез в [github.com](https://github.com) с твоя акаунт.
2. Натисни бутона **New repository** (или "+" горе вдясно → New repository).
3. Въведи име на репозиторието (например: `bom-viewer`).
4. По желание добави описание.
5. Избери дали да е Public или Private.
6. НЕ слагай README, .gitignore или лиценз (ще ги добавим локално).
7. Натисни **Create repository**.

## 2. Инициализиране на git в локалната папка
1. Отвори терминал/PowerShell в папката на проекта (`c:\Users\user\Desktop\Test\Тест 13`).
2. Изпълни следните команди:
   ```sh
   git init
   git add .
   git commit -m "Начален комит: работещ BOM визуализатор"
   ```

## 3. Свързване с GitHub репозитори
1. Копирай адреса на новото репозитори от GitHub (например: `https://github.com/ТВОЕТО_ПОТРЕБИТЕЛСКО_ИМЕ/bom-viewer.git`).
2. В терминала изпълни:
   ```sh
   git remote add origin https://github.com/ТВОЕТО_ПОТРЕБИТЕЛСКО_ИМЕ/bom-viewer.git
   git branch -M main
   git push -u origin main
   ```

## 4. Проверка
- Отиди в GitHub и презареди страницата на репозиторието – ще видиш всички файлове качени.

## 5. Промяна на съобщението на началния commit

### Ако commit-ът е само локален (НЕ е push-нат):
1. Изпълни:
   ```sh
   git commit --amend -m "Новото съобщение за началния комит"
   ```
2. След това можеш да продължиш с push:
   ```sh
   git push -u origin main
   ```

### Ако commit-ът вече е качен в GitHub (вече си правил git push):
1. Изпълни:
   ```sh
   git commit --amend -m "Новото съобщение за началния комит"
   ```
2. След това трябва да форсираш push-а:
   ```sh
   git push -f origin main
   ```
**Внимание:** Ако някой друг работи по това репозитори, форсираният push може да създаде конфликти. Ако си единствен – няма проблем.

## 6. Качване на нови файлове във вече съществуващо GitHub репозитори

1. Увери се, че си в папката на проекта (например: `c:\Users\user\Desktop\Test\Тест 13`).
2. Добави новите файлове към git:
   ```sh
   git add github_repos_vs_projects.md инструкции_за_качване_в_github.md
   ```
3. Направи commit с подходящо съобщение:
   ```sh
   git commit -m "Добавени инструкции и обяснения за GitHub (md файлове)"
   ```
4. Качи (push) промените в GitHub:
   ```sh
   git push
   ```

## 7. Обновяване на вече качен файл в GitHub

Ако имаш промени по даден файл (например .md файл) в локалната си папка и искаш да обновиш версията му в GitHub:

1. Увери се, че си в папката на проекта.
2. Добави променения файл към git:
   ```sh
   git add инструкции_за_качване_в_github.md
   ```
3. Направи commit с описание на промяната:
   ```sh
   git commit -m "Обновени инструкции в инструкции_за_качване_в_github.md"
   ```
4. Качи (push) промяната в GitHub:
   ```sh
   git push
   ```

Това ще обнови файла в GitHub с последната му версия от твоя компютър.

**Забележка:** Всеки път, когато добавяш нови файлове или правиш промени, използвай git add, git commit и git push, за да ги качиш в GitHub.

## 8. Промяна на commit само за определени файлове (например само .md файлове)

### Ако commit-ът с .md файловете е последният (най-горният):
- Можеш да го редактираш с:
  ```sh
  git commit --amend
  ```
- Това ще ти отвори редактор, където можеш да промениш само съобщението на този commit.
- Ако искаш да добавиш/премахнеш файлове от този commit, първо направи нужните промени и ги добави с `git add`, после изпълни `git commit --amend`.

### Ако commit-ът с .md файловете НЕ е последният (имаш по-нови комити след него):
1. Виж историята с:
   ```sh
   git log --oneline
   ```
2. Броиш колко комита назад е този, който искаш да редактираш (например 3 назад).
3. Стартирай интерактивен rebase:
   ```sh
   git rebase -i HEAD~3
   ```
4. В отворения списък смени `pick` на `edit` пред комита, който искаш да редактираш.
5. Когато процесът спре на този commit, направи нужните промени по файловете и ги добави с:
   ```sh
   git add инструкции_за_качване_в_github.md github_repos_vs_projects.md
   git commit --amend
   ```
6. Продължи ребейса:
   ```sh
   git rebase --continue
   ```
7. Ако вече си push-нал тези комити в GitHub, ще трябва да направиш форсиран push:
   ```sh
   git push -f
   ```

**Важно:**
- Препоръчително е да правиш такива промени само ако работиш сам по репозиторито, защото force push може да създаде проблеми за други хора.
- Ако искаш да промениш само съобщението на commit-а, не е нужно да пипаш файловете – само редактирай съобщението с `git commit --amend` или чрез интерактивен rebase.

## 9. Бързи стъпки за обновяване на вече качен файл/commit в GitHub

1. Направи нужните промени по файла и го запази.
2. В терминала изпълни:
   ```sh
   git add инструкции_за_качване_в_github.md
   git commit --amend
   ```
3. Ако се отвори редактор (например Vim):
   - За запис и изход: натисни `Esc`, напиши `:wq` и натисни `Enter`.
   - Ако е само за четене (Read-Only) и не можеш да редактираш, излез с `:q!` и опитай отново.
4. Качи промяната в GitHub:
   ```sh
   git push -f
   ```

**Забележка:** Тези стъпки презаписват последния commit с новото съдържание на файла.

---

**Забележка:** Ако искаш готови команди с твоя URL, просто го копирай от GitHub и замени в примерите по-горе.
