# Разлика между репозитори и проект в GitHub

## Репозитори (Repository)
- Основната единица за съхранение на код в GitHub.
- Съдържа файловете на проекта, историята на промените (commits), клонове (branches), issues, pull requests и др.
- Всеки репозитори е като отделна папка/архив за конкретен софтуерен проект или ресурс.
- Пример: `bom-viewer`, `my-website`, `python-scripts` и т.н.

## Проект (Project)
- Project в GitHub е инструмент за управление на задачи (Kanban board), който помага да се планира, следи и организира работата по един или повече репозитории.
- Не съдържа код, а само табла (boards), колони и карти (cards) за задачи, bug-ове, идеи и др.
- Може да обхваща един или няколко репозитории.
- Използва се за визуално управление на работния процес (например: To Do, In Progress, Done).
- Пример: Project board за следене на всички задачи по няколко свързани репозитории.

---

**Обобщено:**
- **Репозитори** = кода и историята му.
- **Проект** = табло за управление на задачи, свързани с един или повече репозитории.

Можеш да имаш много проекти (boards) за един репозитори или един проект за много репозитории!
